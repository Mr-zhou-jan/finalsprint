## 1. 产品概述

### 1.1 产品目标
FinalSprint 是一个面向大学生的 **AI 期末考试提分系统**。唯一目标：

> 帮助基础薄弱的大学生，在考前有限时间（3天~3周）内，通过上传课程资料、往年题和老师重点，获得一条"最短提分路线"，最终达到目标分数。

### 1.2 目标用户

| 用户画像 | 特征 | 目标分 | 可用时间 |
|----------|------|--------|----------|
| **保命型** | 零基础或极弱，一学期没听课 | 60 分（不挂科） | 考前 3~10 天，每天 1~2h |
| **补救型** | 听过一点但不系统，资料很多不知看什么 | 70~80 分 | 考前 1~3 周，每天 1~3h |
| **速冲型** | 基础一般，想高效刷分 | 80+ 分 | 考前 1~2 周，每天 2~4h |

### 1.3 核心使用场景
1. **考前恐慌**：离考试 5 天，什么都没学，上传老师划的重点和往年题，系统告知"先学这 8 个考点能保 60"
2. **资料爆炸**：手里有教材、PPT、往年题，不知从哪开始，系统给出优先级排序
3. **计划缺失**：每天学 2 小时但不知道学什么最值分
4. **盲目刷题**：做了很多题但分数不涨，系统定位薄弱点并只给薄弱点出题

### 1.4 用户痛点

| 痛点 | 现状 | FinalSprint 解法 |
|------|------|------------------|
| 不知道考什么 | 翻教材目录猜，问学长 | AI 分析多份资料交集提取高频考点 |
| 不知道先学什么 | 从第一章翻到第五章放弃 | ROI 排序：单位时间提分最高的排前面 |
| 不知道学到哪了 | 感觉学了很多但不知道能考几分 | 每次练习后更新估分区间 |
| 不知道今天做什么 | 打开教材发呆 | 每日任务清单 |
| 错题不会处理 | 错了看答案，下次还错 | 自动归因 + 同类题强化 |

### 1.5 产品核心价值
产品回答 5 个问题：
1. **这门课最可能考什么？** → 考情分析页
2. **哪些考点最值得我现在学？** → ROI 排序考点列表
3. **如果我只想保 60，应该先拿下哪些点？** → 保 60 路线
4. **如果我想冲 75/85，应该怎么加码？** → 冲 75/85 路线
5. **我今天这 2 小时具体该干什么最赚分？** → 今日任务

## 2. 用户主流程

### Step 1：创建考试项目
- 用户：登录 → 点击「新建项目」→ 填写科目名、考试日期、目标分（60/75/85）、当前基础、每日时长
- 系统：创建 Project 记录，计算剩余天数
- 输入：`{ subjectName, examDate, targetScore, currentLevel, dailyMinutes }`
- 输出：Project 创建成功 → 跳转战争面板

### Step 2：上传资料
- 用户：进入资料中心 → PDF 上传 或 文本粘贴 → 标注资料类型
- 系统：保存文件/文本 → 创建 Material（status:pending）→ 触发 AI 解析
- 输入：PDF 文件/文本内容 + 标题 + 类型
- 输出：Material 记录 + 跳转审查页

### Step 3：AI 解析资料
- 用户：在审查页等待解析 → 查看 AI 提取的章节/考点/公式 → 可编辑修正 → 确认入库
- 系统：PDF 提取文本 → AI 结构化提取 → 返回 JSON → 用户确认后写 DB
- 输入：原始文本 → AI prompt → LLM
- 输出：`{ materialSummary, chapters[], possibleExamPoints[] }`

### Step 4：生成考情分析（Score Engine）
- 用户：在战争面板点击「生成考点」
- 系统：汇总所有资料的 extractionJson → AI 合并去重 → 生成 15~30 个 ExamPoint（含考试概率、分值、ROI、目标层级）
- 输入：多份 MaterialExtraction + 项目目标参数
- 输出：ExamPoint 列表入库

### Step 5：水平诊断
- 用户：在战争面板点击「开始诊断」→ 做 10~20 道题 → 提交
- 系统：从高 ROI 考点抽题 → AI 批改 → 生成诊断报告（估分区间 + 薄弱点排序 + 推荐优先学习）
- 输入：`{ questionId: userAnswer }[]`
- 输出：DiagnosticReport（predictedMin/Max + strengths + weaknesses + recommendedFocus）

### Step 6：生成冲刺计划
- 用户：点击「生成计划」
- 系统：基于剩余天数 + 目标分 + 诊断结果 + ExamPoint ROI → AI 生成三档每日计划
- 输出：SprintPlan（dailyPlans：每天学什么/做几题/预计提分）

### Step 7：每日学习（核心循环）
```
今日任务 → 学考点（冲刺卡） → 做题 → 批改反馈 → 错题归因 → 同类再练 → 估分更新
```

### Step 8 & 9：模拟考试 + 考前冲刺
- 限时模拟考 → 估分报告
- 考前 24h：高价值错题重做 + 必背公式速查

## 3. 页面级 PRD

### 3.1 战争面板（War Room）
**路由**：`/projects/[id]/overview`
**目标**：3 秒内知道"现在在哪、下一步做什么、距目标还差多少"

#### 模块 1：核心指标行（4 卡片）
| 卡片 | 字段 | 空状态 |
|------|------|--------|
| 目标分数 | targetScore | "--" |
| 当前估分 | predictedMin ~ predictedMax | "待诊断" |
| 分数缺口 | targetScore - predictedMax | "待诊断" |
| 已识别考点 | examPoints.length | "0" |

#### 模块 2：4 步引导卡
| 步骤 | 标题 | 完成条件 | 未完成按钮 | 已完成按钮 |
|------|------|----------|------------|------------|
| Step 1 | 上传资料 | materials.length > 0 | 「上传资料」→ upload | 「管理资料」→ upload |
| Step 2 | 考情分析 | examPoints.length > 0 | 「生成考点」→ analysis | 「查看考点」→ analysis |
| Step 3 | 水平诊断 | diagnosticRuns.length > 0 | 「开始诊断」→ diagnostic/start | 「查看报告」→ diagnostic/report |
| Step 4 | 冲刺计划 | sprintPlans.length > 0 | 「生成计划」→ plan | 「查看计划」→ plan |

步骤间有依赖锁定（前一步未完成则后续 disabled + opacity 50%）

#### 模块 3：今日任务卡片
有任务时：显示今日学什么/做几题/耗时 → 可点击进入考点冲刺卡
空状态："冲刺计划尚未生成，完成诊断后生成"

#### 模块 4：快速入口
考点列表 / 刷题练习 / 错题复盘

### 3.2 科目列表页
**路由**：`/projects`
**目标**：快速查看所有项目，知道每个项目的紧急程度和进度

每个项目卡片：科目名 + 目标分 + 剩余天数 + 估分 + 资料数
空状态：「还没有考试项目」+ 「创建项目」按钮

---

### 3.3 资料上传页
**路由**：`/projects/[id]/materials/upload`
**目标**：上传 PDF 或粘贴文本

- PDF Tab：标题（Input）+ 文件选择 + 「上传并解析」按钮
- 文本 Tab：标题（Input）+ 类型选择（复习资料/往年题/老师重点）+ 文本域 + 「保存并解析」按钮
- 底部：已有资料列表（标题 + 类型 Badge + 状态 Badge）

---

### 3.4 资料解析审查页
**路由**：`/projects/[id]/materials/[materialId]/review`
**目标**：查看/编辑 AI 解析结果，确认入库

| 状态 | 展示 |
|------|------|
| extracting | 骨架屏 + "AI 正在分析中…" |
| failed | Error Alert + 「重试」按钮 |
| reviewed | 解析结果只读展示 |
| 待确认（初次完成） | 可编辑 + 「确认并入库」按钮 |

- 解析结果展示：资料摘要 + 章节列表（关键概念 + 公式 + 老师强调）+ 可能考点列表
- 用户操作：每个字段可编辑 + 「确认并入库」/「重新解析」

---

### 3.5 考情分析页
**路由**：`/projects/[id]/analysis`
**目标**：展示"考什么、最值钱、三档路线分别要什么"

- 三档路线 Tab：保60 / 冲75 / 冲85（按 targetTier 筛选考点）
- 考点列表（可排序）：标题 + 章节 + 考频（进度条）+ 分值 + 耗时 + ROI Badge + 重要性星级 + 掌握状态 + 证据 Tags
- 排序：按 ROI 降序（默认）/按考频/按掌握度升序（薄弱优先）
- 全局统计：考点总数 + 已掌握/学习中/薄弱/未测分布
- 空状态："尚未生成考点，请先上传资料并完成解析"
- 点击考点 → 跳转冲刺卡页

---

### 3.6 考点冲刺卡页
**路由**：`/projects/[id]/points/[pointId]`
**目标**：对一个考点完成"学习→理解→练习→掌握"

五段固定结构：
1. **怎么考**：常见题型 Tags + 常见分值 + 典型问法
2. **一页速通**：最小必要概念 + 必背公式（KaTeX 渲染）+ 做题识别信号
3. **解题模板**：有序步骤列表
4. **高频陷阱**：红色警告卡片（陷阱描述 + 为什么会犯 + 怎么避免）
5. **例题+练习**：例题可展开分步解答 → 「开始练习」出 3~5 题 → AI 批改 → 全部通过 = 标记已掌握

按钮：「开始练习」「标记已掌握」「加入今日任务」

空状态："该考点暂无冲刺卡内容" + 「生成冲刺卡」按钮

---

### 3.7 刷题页
**路由**：`/projects/[id]/practice`
**目标**：选择模式 → 刷题 → 即时批改

- 模式选择：按考点刷/按今日任务/随机/错题重刷
- 刷题会话页（`/practice/session/[sessionId]`）：
  - 进度条（第 X/N 题）+ 题目 + 提交 → 即时反馈
  - 正确：绿色卡片 + 解题思路
  - 错误：红色卡片 + 你的 vs 正确 + 分步解析 + 「加入错题本」
- 完成页：正确率 + 分考点得分 + 估分变化 + 推荐下一步
- 空状态："该考点暂无题目" + 「生成题目」按钮

---

### 3.8 错题本页
**路由**：`/projects/[id]/review`
**目标**：查看错题 + 重做 + 归因

- 统计：总错题数 + 按考点分组 + 未解决数
- 列表（按考点分组，时间倒序）：
  - 题目 + 你的答案 vs 正确答案（红/绿对比）+ 解析
  - 错因标签（概念不清/计算错误/审题失误/完全不会）
  - 重做次数 + 是否已解决
- 操作：「重做」「同类题再练」「标记已解决」
- 空状态："还没有错题 🎉" + 「去刷题」

---

### 3.9 模拟考试页
**路由**：`/projects/[id]/exam`
**目标**：限时模拟 + 估分

V1 简化版：
- 说明页（`/exam/start`）：试卷结构 + 「开始考试」
- 作答页（`/exam/take`）：倒计时 + 题目列表 + 「交卷」（需二次确认）
- 报告页（`/exam/report`）：总分 + 估分 + 各题型正确率 + 用时分析 + 推荐复习

---

### 3.10 冲刺计划页
**路由**：`/projects/[id]/plan`
**目标**：从今天到考试日每天任务清单

- 三档 Tab：保60/冲75/冲85
- 每日卡片：Day N + 日期 + 目标 + 预计提分 + 任务列表 + 完成状态
- Day 详情页（`/plan/day/[day]`）：当天完整任务 + 点击任务进入考点卡/练习
- 空状态："尚未生成计划" + 「去诊断」

## 4. 核心业务规则

### 4.1 项目创建
- 科目名 ≥ 2 字符；考试日期必须在未来；目标分 1-100
- 当前基础：zero/weak/average；每日时长：30/60/120/180 min
- 每用户最多 10 个活跃项目

### 4.2 资料上传与解析
- 状态：`pending → extracting → reviewed` 或 `→ failed → pending（重试）`
- PDF ≤ 20MB；文本 ≤ 50000 字符；每项目 ≤ 20 份资料
- 只有 reviewed 状态的资料参与 ExamPoint 生成

### 4.3 AI 解析入库
- JSON 通过 Zod 校验；失败重试 ≤ 3 次；3 次全失败 → status=failed
- 用户手动编辑结果优先级高于 AI 原始结果

### 4.4 诊断组卷
- 从 importanceLevel≥3 + targetTier 匹配的考点抽题
- 选择题 60% + 填空 25% + 计算 15%
- 难度：基础 40% + 中等 40% + 困难 20%
- 总题数 10~20

### 4.5 提分计划生成
- 每天任务时长 ≤ dailyMinutes
- 优先安排 ROI 高 + masteryStatus=weak 的考点
- 每 2 考点学完 → 1 练习；每 3 天 → 1 复盘
- 考前最后 1 天 → 仅错题重做 + 公式速查

### 4.6 错题归因
- V1：用户自选错因（概念不清/计算错误/审题失误/完全不会）
- 连续 3 次错 → 降级为 weak；连续 3 次对 → 升级为 good

### 4.7 分数预测
- predictedScore = Σ(masteryWeight × point.estimatedScore) / Σ(point.estimatedScore) × 100
- 权重：unknown=0.1, weak=0.3, pass=0.65, good=0.9
- 估分区间 = [预测分-8, 预测分+5]

## 5. MVP 范围

### V1 必须做（P0）
注册/登录、项目 CRUD、PDF+文本上传、AI 资料解析、ExamPoint 生成+ROI、
考情分析页、诊断系统（组卷+作答+报告）、冲刺计划生成+展示、
考点冲刺卡（AI 生成+五段式）、练习系统（按考点/按计划+AI 批改）、
错题本（收集+分组+重做+同类题）、估分更新、战争面板

### V1 不做（延后 V2/V3）
视频链接解析、图片 OCR、PPT 解析、完整考试模拟（V1 简化版）、
知识图谱可视化（先用列表）、多 AI Provider 智能调度（先用 DeepSeek）、
移动端适配、社交/打卡/排行榜

### V1 成功指标
| 指标 | 目标 |
|------|------|
| 7 日留存 | ≥ 40% |
| 资料上传完成率 | ≥ 80% |
| 诊断完成率 | ≥ 60% |
| 平均估分提升 | ≥ 8 分 |
| NPS | ≥ 30 |

## 6. 指标体系

### 激活指标
- 注册转化率 ≥ 25%
- 项目创建率 ≥ 70%
- 首次上传率 ≥ 60%
- 激活时间 ≤ 15min

### 核心使用指标
- DAU 基数×30%
- 人均日学习时长 ≥ 45min
- 每日任务完成率 ≥ 50%
- 人均日练习题 ≥ 15 题

### 学习完成指标
- 考点掌握率 ≥ 70%（考前）
- 错题解决率 ≥ 80%（考前）
- 计划坚持率 ≥ 50%

### 提分指标（最核心）
- 平均估分提升 ≥ 8 分
- 估分达标率（估分 ≥ 目标分）
- 首日提分
- 最后一周提分趋势

### 留存指标
- 次日留存 ≥ 60%
- 7 日留存 ≥ 40%
- 考前留存 ≥ 50%
- 多科目复用率 ≥ 30%
